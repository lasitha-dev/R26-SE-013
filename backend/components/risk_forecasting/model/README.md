# FMD Outbreak Prediction Model — Sri Lanka (2017–2024)

This repository contains the research code and an interactive dashboard for predicting Foot-and-Mouth Disease (FMD) outbreaks across 25 districts in Sri Lanka, using machine learning techniques on environmental, spatial, and meteorological data.

## 🏆 Key Achievements & Implementations

*   **Predictive Modeling Pipeline:** End-to-end machine learning pipeline utilizing walk-forward validation optimized for temporal dependencies.
*   **Feature Engineering:** Engineered 21 core features, including cyclical time encoding, monsoon phases, and weather lags.
*   **Model Performance:** The Stage 1 model achieved a **ROC-AUC of 0.843**, with **Recall: 76.9%**, **Precision: 52.6%**, and **F1-Score: 0.625**.
*   **Interactive Streamlit Dashboard:** Developed a comprehensive, multi-page web application to interactively explore data, visualize model performance, generate district forecasts (including climatological forecasting), and provide model explainability.
*   **Advanced Analytics:** Implemented bootstrap uncertainty quantification for robustness evaluation.
*   **Automation Scripts:** Added Python scripts for seamless model retraining and automated plot regeneration.

## 🚀 How to Run the Project

### Research Notebooks
To reproduce the research, open and run the notebooks in sequential order:
1. **`01_Data_Collection_and_Understanding.ipynb`**: Explores the raw data sources (DAPH, CHIRPS, GLW, Admin boundaries).
2. **`02_Data_Preprocessing_and_Feature_Engineering.ipynb`**: Details data merging and feature engineering.
3. **`03_Exploratory_Data_Analysis.ipynb`**: Analyzes imbalances, outbreak distributions, and feature correlations.
4. **`04_Model_Training_and_Evaluation.ipynb`**: Evaluates models using walk-forward time-series cross-validation.
5. **`05_Final_Model_and_Predictions.ipynb`**: Trains the final chosen model and generates predictions and probability heatmaps.
6. **`06_Interactive_Prediction.ipynb`**: Provides interactive prediction utilities and testing.
7. **`07_Stage2_Severity_Score_and_Model.ipynb`**: Implements the Stage 2 model focusing on outbreak severity scoring.
8. **`08_SHAP_Explainability.ipynb`**: Analyzes feature importance and uses SHAP values for model explainability.
9. **`09_Bootstrap_Uncertainty.ipynb`**: Applies bootstrap techniques to quantify model uncertainty and ensure robustness.

### Streamlit Dashboard
To launch the interactive dashboard, run the following command in your terminal from the root directory:
```bash
pip install -r app/requirements.txt
streamlit run app/streamlit_app.py
```

## 📊 Evaluation Strategy

Because FMD outbreaks are temporally dependent, random train-test splitting is inappropriate. The models were evaluated using **expanding-window walk-forward validation**:
* Train 2017-2021 → Test 2022
* Train 2017-2022 → Test 2023
* Train 2017-2023 → Test 2024
